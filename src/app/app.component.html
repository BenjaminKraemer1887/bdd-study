Gherkin

Gherkin is designed to be easy to read and understand, making it accessible to all stakeholders, including those without technical expertise.

It uses a specific syntax to structure the specifications, with keywords like Feature, Scenario, Given, When, Then, And, and But.

Case Study

Userstory: As a registered user, I want to log in to the web application So that I can access my personalized dashboard
Description: The login functionality should allow registered users to enter their credentials (username and password) and access their personalized dashboard if the credentials are correct. If the credentials are incorrect, the user should be notified with an appropriate error message.
Acceptance Criteria: Successful login with valid credentials:
The user should be able to enter their username and password. Upon entering valid credentials and clicking the "Login" button, the user should be redirected to their personalized dashboard. The user should see a welcome message that includes their name on the dashboard. Unsuccessful login with invalid credentials:
If the user enters an invalid username or password, they should not be redirected. An error message should be displayed, indicating that the username or password is incorrect. Unsuccessful login with missing credentials:
If the user tries to log in without entering a username or password, they should remain on the login page. An error message should be displayed, indicating that the username and password are required fields.
